LOCAL STORAGE AND SESSION STORAGE

LOCAL STORAGE
-Browser based storage, where we storage our data in the form of keys and values
-After refreshing our page, that data will be available
-Data is stored in the form of string format
-Data size approx 5MB
-storing the data permanently
------------------------------------------------------------------------------------------------------------------------------------------------------------------------
METHODS IN LOCAL STORAGE
-getitem() - get a particular data from the storage
-setitem() - insert a partiicular item and store them , Stores data in the browser's localStorage under a specified key.
-removeitem() - remove particular item from local storage
-clearitem() - remove whatever item from local storage
-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
JSON.stringify() - converts js object to json string

.parse - json string back to js object
------------------------------------------------------------------------------------------------------------------------------------------
SESSION STORAGE
-approach in browser , that help us to store data in browser temperorily
-data is stored in the form of key and value
-Its only available within the current SESSION
-If we refresh the page, next session the data is lost 
-Storage size 5MB
--------------------------------------------------------------------------------------------------------------------------------------------------------
When to use SessionStorge and LocalStorage?
when we need data even after refreshing the page - local storage
When we do not want data after refreshing page - session storage
