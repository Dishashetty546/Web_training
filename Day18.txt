Redux?
- library which is used to manage state of our application
- Data warehouse, all the data is stored and managed in a predictable way
-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
Why do we need?
Mainly used to 
all state is controlled centered 
easy to use in large scale
--------------------------------------------------------------------------------------------------------------------------------------------------------------------------
when?
there are multiple components, all components are sharing same data
when we want to manage complex data flow
when we are building large application
predictable and testable states

when not to use?

components of redux?
store - holds entire data of our application
action - discribes what happened
reducer - decides how to change the state/data
Dispatch - sends the action to the reducer
UI(Subscriber) - UI react component which manages state/data
-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
